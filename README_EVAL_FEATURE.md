# 模型评估功能说明

## 新增功能概述

在原有的Streamlit应用中新增了**模型评估功能**，允许用户通过Web界面直接配置和运行模型评估，无需手动执行命令行脚本。

## 主要特性

### 1. 模型配置
- **模型名称**: 支持各种OpenAI兼容的模型（如claude-sonnet-4-20250514-thinking, gpt-4o等）
- **API Key**: 安全的密码输入框
- **Base URL**: 支持自定义API端点（如代理API）
- **Max Tokens**: 控制模型响应的最大token数（1-32768）
- **Temperature**: 控制模型输出的随机性（0.0-2.0）

**注意**: 所有配置字段都需要手动填写，不再提供默认值

### 2. 测试配置
- **特定场景-任务**: 选择特定场景（indoor/outdoor）和任务类型，设置测试案例数（1-30）
- **全部场景-任务**: 测试所有场景和任务类型，设置每个场景的测试案例数（1-30）

### 3. 实时监控
- **进度条**: 显示当前评估进度
- **Prompt文件路径**: 显示生成的prompt文件路径
- **停止评估**: 提供停止评估的按钮
- **状态管理**: 自动处理运行、完成、错误等状态

**注意**: 为了简化界面，不再显示详细的评估日志输出

### 4. 任务类型支持
- MotionState（运动状态推理）
- GeoLocation（地理位置推理）
- ArrivalTimeInterval（到达时间间隔推理）
- CausalReordering（因果重排序推理）
- TrajectoryForecasting（轨迹预测）
- NextSpotForecasting（下一位置预测）
- MultiTargetTrajectoryForecasting（多目标轨迹预测）

## 使用方法

### 1. 启动系统
```bash
./run.sh
```

### 2. 访问评估功能
1. 在侧边栏点击"🚀 启动模型评估"
2. 配置模型参数和测试范围（所有字段都需要填写）
3. 点击"🚀 开始评估"
4. 实时查看评估进度和prompt文件路径
5. 如需停止，点击"⏹️ 停止评估"

### 3. 查看结果
- 在侧边栏点击"📈 查看评估结果"查看详细分析
- 直接从`./eval/results/`文件夹中选择JSON结果文件
- 支持查看完整的题目内容，包括地图和摄像头图像
- 显示模型回答、提取答案、正确答案和评估指标
- 结果文件自动保存到`eval/results/`目录

## 技术实现

### 1. 后台执行
- 使用`subprocess`在后台线程中执行`eval.py`
- 实时捕获和显示输出信息
- 自动解析进度信息更新进度条

### 2. 状态管理
- 使用Streamlit的session_state管理评估状态
- 支持运行中、完成、错误等状态切换
- 自动刷新界面显示最新状态

### 3. 参数传递
- 通过命令行参数将UI配置传递给`eval.py`
- 支持所有原有参数：model, api-key, base-url, max-tokens, temperature等

## 注意事项

1. **API配置**: 确保API Key有效且有足够额度
2. **网络连接**: 确保能够访问指定的Base URL
3. **资源消耗**: 大量测试案例可能需要较长时间和较多API调用
4. **并发限制**: 建议根据API限制合理设置测试数量

## 故障排除

1. **评估失败**: 检查API Key和网络连接
2. **进度卡住**: 可能是API限制，等待或减少测试数量
3. **结果不显示**: 检查`eval/results/`目录权限
